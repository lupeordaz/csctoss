Description

In order to stop using JBilling, we need to modify the following system components.

Provisioning Tool We only need to revise the following features on provisioning tool.
Activate Multiple Devices
Load Devices
Assign Device
Database functions
Current provisioning tool gets customer information and sales order number (SO-xyz) from JBilling, but once we stopped using JBilling, we cannot get sales order number which some customers still need for their billing, so we need to keep sales order number (SO-xyz) in OSS.

How current provisioning tool works
Current provisioning tool retrieves information from JBilling as follows.

Activation
Retrieve "Pending Approval" sales order numbers (SO-xyz) and its customer name from JBilling.
Retrieve shipping information from JBilling
Name
Phone Number
Email, Address
Shipping Method [FedEx 2nd Day Air], Product Code [E,g. IPG-7310], Quantity).
Look for MRC product code based on a selected sales order number (SO-xyz) in JBilling.
Insert MRC product code and sales order number (SO-xyz) into plan table in OSS.
Update prov_line record in Jbilling to activate.
Revised version of Provisioning tool should be (No JBilling)
Activation
Select customer name.
Enter sales order number ("SO-xyz" and "WS-xyz" and "GP order number") manually
Select Product Code (E.g. IPG-7310) and quantity
Select MRC product code and quantity
Insert MRC product code and sales order number ("SO-xyz" and "WS-xyz" and "GP order number") into plan table in OSS, so that OSS knows which line is tied to which sales order number.
