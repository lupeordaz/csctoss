Git Notes  


    


Clone a Repository
    
  This will create the staging area for retrieving/adding the code of record.  
  Literally, we are checking out the project.

  Get the link to the Gitlab repository from the Gitlab website.  For this download,
  use the Git repository Webapp / cnos.  Once in the repository in Gitlab, highlight
  the SSH address for the repository.  

  Go to the directory on Centos where you will store the repository (usually /home/gordaz).

  Issue the following command:

  git clone git@jcigit01.jcigroup.local:webapp/cnos.git 

Display Git status
     
    
  After cloning the work area, you now have a staging area for controlling
  your code.  You can modify your entries in this work area.  

  The git status option lets you see where in the Git process your
  code is at.  Executing the status shows the following:

    [gordaz@cctlix03 opentaps_transaction_scripts]$ git status
    # On branch master
    # Changes not staged for commit:
    #   (use "git add ..." to update what will be committed)
    #   (use "git checkout -- ..." to discard changes in working directory)
    #
    # modified:   get_active_lines_from_jbill_and_oss_for_opentaps.sql
    # modified:   get_cancelled_lines_from_jbill_and_oss_for_opentaps.sql
    #
    no changes added to commit (use "git add" and/or "git commit -a")

  The status shows the modules that have been modified.  To add these modified
  modules to the branch, they must first be committed as indicated by the 
  message. 

Git Add function
    
  Issuing the 'git add' command will stage the changes made to the 'Commit'
  stage.  

    [gordaz@cctlix03 opentaps_transaction_scripts]$ git add get_active_lines_from_jbill_and_oss_for_opentaps.sql
    [gordaz@cctlix03 opentaps_transaction_scripts]$ git add get_cancelled_lines_from_jbill_and_oss_for_opentaps.sql
    [gordaz@cctlix03 opentaps_transaction_scripts]$ git status
    # On branch master
    # Changes to be committed:
    #   (use "git reset HEAD ..." to unstage)
    #
    # modified:   get_active_lines_from_jbill_and_oss_for_opentaps.sql
    # modified:   get_cancelled_lines_from_jbill_and_oss_for_opentaps.sql
    #

  The modules are now ready for the 'commit' stage.
